# 第一阶段功能测试与验收

本文档用于验证当前阶段已完成能力：Gazebo 仿真启动、阿克曼转向控制、里程计、TF、传感器、EKF 定位输出，以及 Nav2/LIO-SAM2 启动预检查。

执行目录：

```bash
cd /home/xavier/Workspace/ClaudeSpace/ros2_robot_sim
source /opt/ros/humble/setup.bash
source install/setup.bash
```

## 0. 构建验证

目的：确认 ROS 2 包能正常构建，安装资源可被发现。

```bash
colcon build --packages-select robot_description
source install/setup.bash
ros2 pkg list | grep robot_description
```

通过标准：

- `colcon build` 无错误。
- 输出包含 `robot_description`。

## 1. 启动仿真

先清理旧进程：

```bash
pkill -f "ros2 launch robot_description robot_simulation.launch.py" || true
pkill -f "gzserver" || true
pkill -f "gzclient" || true
pkill -f "rviz2" || true
```

启动 Gazebo 和 RViz：

```bash
ros2 launch robot_description robot_simulation.launch.py
```

通过标准：

- Gazebo 加载 `corridor_tunnel.world`。
- 机器人 `ackermann_robot` 成功 spawn。
- RViz 正常启动。
- 终端出现以下关键信息：

```text
Subscribed to [/robot/cmd_vel]
Advertise odometry on [/robot/odom]
Advertise steerangle on [/robot/steerangle]
```

## 3. 阿克曼直行测试

发送 3 秒直行命令：

```bash
timeout 3s ros2 topic pub -r 20 /robot/cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.5}, angular: {z: 0.0}}"
```

检查 odom：

```bash
ros2 topic echo /robot/odom --once
```

通过标准：

- Gazebo 中车辆向前运动。
- `/robot/odom` 中 `pose.pose.position.x` 发生明显变化。
- 车辆不原地旋转，不出现明显侧滑。

## 4. 阿克曼转向测试

发送 3 秒转向命令：

```bash
timeout 3s ros2 topic pub -r 20 /robot/cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.5}, angular: {z: 0.25}}"
```

检查转角：

```bash
ros2 topic echo /robot/steerangle --once
ros2 topic echo /joint_states --once
```

通过标准：

- Gazebo 中车辆沿弧线运动。
- `/robot/steerangle` 有非零值，通常约 `0.2 rad` 左右。
- `/joint_states` 中 `left_steering_joint` 和 `right_steering_joint` 有非零转角。
- 左右前轮转角不同，说明阿克曼内外轮角差生效。

## 5. Odom 频率测试

```bash
ros2 topic hz /robot/odom
```

观察 5 到 10 秒后按 `Ctrl+C`。

通过标准：

- 有稳定输出。
- 频率通常为几十 Hz，受 Gazebo 实时率影响，不要求严格等于 50 Hz。

## 6. TF 测试

```bash
ros2 topic echo /tf --once
```

通过标准：

- 能看到 `odom -> base_footprint`。
- RViz Fixed Frame 设置为 `odom` 时不报 fixed frame 错误。
- RobotModel 能正常显示。

可选生成 TF 图：

```bash
ros2 run tf2_tools view_frames
```

通过标准：

- TF 树包含 `odom`、`base_footprint`、`base_link`、`laser_link`、`imu_link`、`gps_link`。
- TF 树包含四个轮子和前转向 link。

## 7. 传感器测试

IMU：

```bash
ros2 topic echo /robot/imu/data --once
```

点云：

```bash
ros2 topic echo /robot/velodyne_points --once --field width
ros2 topic hz /robot/velodyne_points
```

GPS：

```bash
ros2 topic echo /robot/rtk_gps/fix --once
```

通过标准：

- IMU 有 `orientation`、`angular_velocity`、`linear_acceleration`。
- 点云 `width` 大于 0。
- `/robot/velodyne_points` 有稳定频率。
- GPS topic 有数据；隧道/室内场景下 GPS 精度不作为主失败项。

## 8. EKF 定位测试

仿真运行时另开终端：

```bash
ros2 launch robot_description localization.launch.py
```

再开一个终端检查：

```bash
ros2 topic list | grep odometry
ros2 topic echo /odometry/filtered --once
```

通过标准：

- topic 列表包含 `/odometry/filtered`。
- `/odometry/filtered` 输出 `nav_msgs/msg/Odometry`。
- `frame_id` 为 `odom`。
- `child_frame_id` 为 `base_footprint`。

## 第一阶段通过判定

主功能通过条件：

- 阿克曼车模型能在 Gazebo 中稳定生成。
- `/robot/cmd_vel` 能控制车辆直行和弧线转向。
- 前轮有真实转向角，左右前轮存在内外轮角差。
- `/robot/odom`、`/tf`、`/joint_states` 正常发布。
- IMU、点云、GPS 话题存在并能输出数据。
- EKF 能发布 `/odometry/filtered`。

扩展预检查通过条件：

- Nav2 缺依赖时能明确报告，不直接崩溃。
- LIO-SAM2 缺依赖时能明确报告，不直接崩溃。
