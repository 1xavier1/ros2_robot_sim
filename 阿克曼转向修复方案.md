# 阿克曼转向修复方案

## 问题诊断

### 症状
- 车辆命名为 `ackermann_robot`，但无法转向
- 前端摇杆/键盘发送 `/robot/cmd_vel` 后车辆仅能直行，不能转弯
- GPS 数据不变化（车辆无法移动的连锁反应）

### 根因分析

项目存在三层架构缺陷：

**1. 关节类型错误（URDF）**

前轮转向关节 `left_steering_joint` 和 `right_steering_joint` 被设为 `type="fixed"`（焊死），无法旋转：

```xml
<!-- 修复前 -->
<joint name="left_steering_joint" type="fixed">
  <parent link="base_link"/>
  <child link="left_steering_link"/>
</joint>
```

**2. 驱动插件不兼容**

- `libgazebo_ros_diff_drive.so`：仅控制后轮前进/后退，没有转向能力
- `libgazebo_ros_ackermann_drive.so`：为 Prius 模型的 **universal 双轴关节**（同一关节同时做转向和驱动）设计，与我们的**分离式关节**（转向关节 Z 轴 + 车轮关节 Y 轴独立）不兼容
- `gazebo_ros2_control` + `ackermann_steering_controller`：在 ROS 2 Humble 中存在 pluginlib 兼容性 bug——`GazeboSystem` 硬件插件注册在 `gazebo_ros2_control__pluginlib__plugin` 资源索引下，而 `ResourceManager` 查找 `hardware_interface__pluginlib__plugin`，导致控制器加载失败

**3. 测试文件主动阻止修复**

`test_wheel_encoder_integration.py` 中显式断言：

```python
assert "libgazebo_ros_ackermann_drive.so" not in urdf
```

---

## 解决方案

### 总体策略

**diff_drive（后轮驱动）+ 自定义 Gazebo 模型插件（前轮转向 PID 控制）**

绕过 Humble 中所有已知的兼容性问题：
- 保留稳定的 `libgazebo_ros_diff_drive.so` 做后轮驱动
- 自研 `libsteering_controller.so` Gazebo 模型插件，使用 Gazebo 内置的 `JointController` API 做 PID 位置控制
- Python `cmd_vel_bridge.py` 桥接 Twist → 后驱 + 阿克曼转向角

### 数据流架构

```
┌─────────────────┐
│  remote_control  │  (摇杆/键盘/WebSocket)
│  Nav2 navigation │  (导航堆栈)
└────────┬────────┘
         │ Twist
         ▼
┌─────────────────────────────────┐
│  /robot/cmd_vel                 │
└────────┬────────────────────────┘
         │
         ▼
┌─────────────────────────────────┐
│  cmd_vel_bridge.py              │  (Python 桥接节点)
│  ─────────────────────────      │
│  linear.x  → /robot/rear/cmd_vel│
│  angular.z → 自行车模型          │
│            → Ackermann 几何      │

└────────┬───────────┬────────────┘

┌──────────────┐ ┌──────────────────────────┐

```

## 修改清单

### 1. `src/robot_description/urdf/robot_base.urdf.xacro`

| 修改项 | 旧值 | 新值 |
|--------|------|------|
| `left_steering_joint` 类型 | `fixed` | `revolute` + Z轴 + ±0.6rad 限位 + dynamics |
| `right_steering_joint` 类型 | `fixed` | `revolute` + Z轴 + ±0.6rad 限位 + dynamics |
| 驱动插件 | `libgazebo_ros_diff_drive.so` (topic: `cmd_vel`) | `libgazebo_ros_diff_drive.so` (remap: `rear/cmd_vel`) |
| 转向插件 | (无) | `libsteering_controller.so` |
| 关节状态发布 | (无) | `libgazebo_ros_joint_state_publisher.so` |

转向关节修复后：

```xml
<joint name="left_steering_joint" type="revolute">
  <parent link="base_link"/>
  <child link="left_steering_link"/>
  <origin xyz="${wheelbase/2} ${track_width/2} 0" rpy="0 0 0"/>
  <axis xyz="0 0 1"/>
  <limit lower="-0.6" upper="0.6" effort="100" velocity="10"/>
  <dynamics damping="0.1" friction="0.1"/>
</joint>
```

驱动插件改为后轮专用话题：

```xml
<plugin name="gazebo_ros_diff_drive" filename="libgazebo_ros_diff_drive.so">
  <ros>
    <namespace>/robot</namespace>
    <remapping>cmd_vel:=rear/cmd_vel</remapping>
  </ros>
  <left_joint>rear_left_joint</left_joint>
  <right_joint>rear_right_joint</right_joint>
  ...
</plugin>
```

新增转向控制器插件：

```xml
<plugin name="steering_controller" filename="libsteering_controller.so">
  <left_steering_joint>left_steering_joint</left_steering_joint>
  <right_steering_joint>right_steering_joint</right_steering_joint>
  <steering_p>50.0</steering_p>
  <steering_i>0.0</steering_i>
  <steering_d>5.0</steering_d>
  <topic>/robot/steering_cmd</topic>
</plugin>
```

### 2. `src/robot_description/src/steering_controller.cpp`（新建）

Gazebo 模型插件，核心实现：

```cpp
void Load(physics::ModelPtr _model, sdf::ElementPtr _sdf) override
{
    // 读取 SDF 参数
    // 获取关节并设置 ODE 力矩上限
    this->left_joint->SetParam("fmax", 0, 100.0);
    this->right_joint->SetParam("fmax", 0, 100.0);

    // 使用 Gazebo 内置 JointController API 做 PID 位置控制
    auto jc = _model->GetJointController();
    jc->AddJoint(this->left_joint);
    jc->AddJoint(this->right_joint);
    jc->SetPositionPID(left_joint->GetScopedName(), common::PID(kp, ki, kd));
    jc->SetPositionPID(right_joint->GetScopedName(), common::PID(kp, ki, kd));

    // 订阅 ROS 话题接收目标角度 [left, right]
    this->sub = this->ros_node->create_subscription<Float64MultiArray>(
        topic, 10, [this](auto msg) {
            jc->SetPositionTarget(left_joint->GetScopedName(), msg->data[0]);
            jc->SetPositionTarget(right_joint->GetScopedName(), msg->data[1]);
        });

    // 每个仿真步调用 Update() 执行 PID
    this->update_connection = event::Events::ConnectWorldUpdateBegin(
        std::bind(&SteeringControllerPlugin::OnUpdate, this));
}
```

| 参数 | 值 | 说明 |
|------|-----|------|
| `steering_p` | 50.0 | PID 比例增益 |
| `steering_i` | 0.0 | PID 积分增益 |
| `steering_d` | 5.0 | PID 微分增益 |
| `fmax` | 100 N⋅m | ODE 关节力矩上限 |

### 3. `scripts/cmd_vel_bridge.py`（新建）

Python 桥接节点：

- 订阅 `/robot/cmd_vel` (Twist)
- 发布 `linear.x` → `/robot/rear/cmd_vel` (Twist)
- 发布阿克曼转向角 → `/robot/steering_cmd` (Float64MultiArray [left, right])
- **自动检测转向控制器就绪**：每 1 秒检查 `/robot/steering_cmd` 的订阅者数量，检测到后才开始发布

转向角计算：

```python
def control_loop(self):
    linear = self.target_linear
    angular = self.target_angular

    if abs(linear) < 0.01:
        target_steer = angular * 0.5   # 原地转向
    else:
        target_steer = math.atan(self.wheelbase * angular / linear)  # 自行车模型

    # 阿克曼几何：内外轮角差
    if abs(target_steer) > 1e-4:
        R = self.wheelbase / math.tan(abs(target_steer))
        left_target  = math.atan(self.wheelbase / (R - track_width / 2.0))
        right_target = math.atan(self.wheelbase / (R + track_width / 2.0))
        if target_steer < 0:
            left_target, right_target = -right_target, -left_target
```

## 已知限制与注意事项

1. **首次 spawn 耗时长**：Gazebo 加载世界 + 生成模型可能需要 30-60 秒，后续启动会更快。`cmd_vel_bridge` 会自动等待转向控制器就绪，无需手动干预。

2. **GAZEBO_PLUGIN_PATH**：必须执行 `source install/setup.bash`，环境钩子会自动设置此变量。如果手动启动 Gazebo，需要手动 export。

3. **转向 PID 调参**：默认 kp=50, kd=5 适用于当前模型物理参数。如需调整，修改 URDF 中的 `<steering_p>` 和 `<steering_d>` 参数后重新 `colcon build`。

4. **Gazebo 11 EOL 警告**：Gazebo Classic 已于 2025 年 1 月 EOL，建议未来迁移至新 Gazebo (Ignition/Garden)。
